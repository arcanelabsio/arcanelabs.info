Software built for reuse.

Small tools, libraries, and apps from a single builder. Every project starts from one principle: solve common problems with the minimum complexity needed and the maximum flexibility to adapt. When an idea graduates past ideation, it ships here.

How I build
01

Contracts before code

Schemas, interfaces, and architectural decisions are written first; implementations follow. Every non-trivial choice lives as an ADR in the project's docs/adr/ — future-me gets the why, not just the what.

02

Small surface per project

One thing done well beats five things half-done. Each project is scoped so a single builder can ship it and support it. Architecture is chosen for the problem, not imposed by an org-wide playbook.

03

AI-augmented, not AI-dependent

I use AI assistants heavily in my own workflow, and I build tooling (Forge, practitioner workflows) that assumes AI is part of the toolkit. But the products degrade gracefully when the model isn't there.

04

Reuse before rebuild

Every new project is scouted for what already exists in the portfolio. drive_sync_flutter is the shape of that bet — one library that solves Drive sync once, used by every app that needs it.
